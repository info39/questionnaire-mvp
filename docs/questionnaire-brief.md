# Questionnaire Brief

## Goal
Build a conversational onboarding questionnaire that helps understand:
- the business
- the users
- the type of website needed
- required features
- overall complexity

---

## Product Thinking

The questionnaire should:
- feel like a guided experience, not a form
- reduce fatigue
- create clarity for both sides
- surface hidden needs
- translate answers into product decisions

---

## Modes

### Mode A (Short)
- faster
- fewer questions
- surface-level understanding

### Mode B (Deep)
- more detailed
- includes follow-up logic
- deeper UX and product insights

---

## Core Outputs

From the answers, we want to understand:

- What type of site this is (informational / ecommerce / hybrid)
- What features are required
- Level of complexity (low / medium / high)
- Content structure needs
- User behavior patterns

---

## UX Requirements

- conversational flow
- progress visibility
- clear steps (like course chapters)
- ability to go back
- feeling of advancement

---

## Technical Target

- WordPress + Elementor
- self-contained component
- embedded in one HTML widget
- RTL support
- responsive
